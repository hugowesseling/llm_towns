# LLM Towns

A medieval fantasy village simulation where AI-driven villagers autonomously pursue goals generated by LLMs. Built as a monolithic Python Flask server with a tick-based simulation loop.

## World Structure

- **Single 2D grid** — towns are built from specialized tiles placed directly on the world grid (Option B: no separate sub-grids).
- **Terrain tiles** — grass, forest, water, mountain, sand, snow, swamp, road, **building**, **house**, **town square**.
- **Towns** — each occupies a rectangular area (8–12 tiles per side) on the world grid with:
  - A cross of N-S and E-W **road** tiles forming main streets
  - A 3×3 **town square** at the center
  - **Building** tiles along the perimeter (market, inn, blacksmith, etc.)
  - **House** tiles in the interior
  - **4 entry/exit points** (north, south, east, west) — road tiles at the midpoint of each side where roads connect
- Buildings are **non-walkable**; villagers path around them via streets and the town square.
- **Roads** connect towns via their entry points, traced between town centers.

## Simulation Loop

Tick-based (1 tick/second). Per tick:

1. **Goal assignment** — idle villagers get new goals from the LLM (or fallback), with town context injected into the prompt
2. **Plan generation** — goals are converted to action sequences by the LLM or deterministic fallbacks
3. **Action execution** — actions progress through their lifecycle; A* pathfinding on the main grid for movement
4. **Trading** — villagers buy/sell at their home town's market
5. **State updates** — needs decay, inventories change, memories accumulate

## Inter-Town Travel

- The **travel** action type moves a villager between towns via A* pathfinding along roads.
- LLM prompt context includes the villager's home town name and nearby towns with their relations — so the LLM can generate goals like *"Visit Stonebrook to trade"*.
- The fallback planner detects town names in goal descriptions and generates travel actions.
- Possible actions expose road-connected towns as travel targets.
- Interaction memories include town affiliations (*"Talked with Elara from Stonebrook"*), so town names propagate through the social graph.

## Module Layout

```
app.py                         Flask server, API, simulation bootstrap
world/world_generator.py       World/town generation, terrain, roads
simulation/scheduler.py        Tick loop, pathfinding, action execution
simulation/planner.py          Goal → plan conversion (LLM + fallback)
simulation/actions.py          Action/Goal/Plan lifecycle
simulation/villager.py         Villager entity model
simulation/market.py           Trading system
simulation/decision.py         Cognitive decision timing
llm/brain.py                   LLM API client
llm/prompts.py                 Prompt builders with town context
```

## Design Pattern

```
Goal (why) → Plan (how) → Actions / sub-actions (what executes) → Atomic ticks (when)
```
